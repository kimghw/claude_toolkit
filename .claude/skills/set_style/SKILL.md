# set_style — 프로젝트 응답 문체 지침 주입

본 스킬은 **Claude 가 사용자에게 답할 때 쓰는 문체** 를 프로젝트 단위로 고정한다. 선택된 프리셋을 프로젝트 루트 `CLAUDE.md` 안에 marker 구간으로 박아두면, 모든 후속 세션 시작 시 `claudeMd` context 로 자동 로드되어 별도 hook 없이 리마인드된다.

> 범위 밖: 사용자 전역 `~/.claude/CLAUDE.md`, 코드 포매터·린터 규약, 산출 문서(.md/.docx) 본문의 문체 변환. 관련 스킬: `/set_toolkit` (스킬별 사전조건·toolignore 통합).

## 핵심 원칙

- **단일 출처**: 문체 지침은 프로젝트 루트 `CLAUDE.md` 의 marker 구간 한 곳에만 존재한다. 다른 곳에 중복 작성·언급하지 않는다.
- **호출마다 대체**: 재호출 시 marker 구간 본문 전체를 새 프리셋으로 **완전 교체** (누적·병합 금지). marker 바깥 영역은 절대 손대지 않는다.
- **프리셋 = 본문 그대로**: 아래 §3 에 정의된 텍스트를 글자 그대로 marker 사이에 쓴다. 사용자가 추가 지시("좀 더 짧게") 를 줘도 본문을 임의로 수정하지 말고, 별도 marker 외 영역에 본인이 직접 적도록 안내한다.
- **사용자 프로젝트 한정**: 본 스킬은 `$CLAUDE_PROJECT_DIR/CLAUDE.md` 만 다룬다. 전역 CLAUDE.md 는 건드리지 않는다.

## 사용 도구

| 도구 | 용도 |
|:---|:---|
| `AskUserQuestion` | 프리셋 4종 + 제거/현황확인 옵션 제시 |
| `Read` | 현재 `CLAUDE.md` 와 marker 상태 확인 |
| `Edit` / `Write` | marker 구간 교체 또는 파일 신규 생성 |

## 1. 절차

### 1.1 인자 분기

호출 형식: `/set_style [개조식|평어체|기획보고서|규정|status|remove]`

| 인자 | 동작 |
|:---|:---|
| (없음) | `AskUserQuestion` 으로 프리셋 선택받음 |
| `개조식` / `평어체` / `기획보고서` / `규정` | 그 프리셋 즉시 적용 (AskUserQuestion 생략) |
| `status` | 현재 `CLAUDE.md` 의 marker 안 본문을 그대로 출력하고 종료 (변경 없음) |
| `remove` | marker 구간 전체(마커 포함)를 제거하고 종료 |

### 1.2 현황 확인

1. `$CLAUDE_PROJECT_DIR/CLAUDE.md` 존재 여부 확인.
2. 존재하면 `<!-- STYLE-BEGIN /set_style -->` 와 `<!-- STYLE-END /set_style -->` 가 **둘 다 있는지** 검사.
   - 둘 다 있음 → 정상 상태. 사이 본문을 추출해 현재 적용 프리셋을 표시(`status` 또는 사용자 안내용).
   - 한 쪽만 있음 → 파일 손상. 사용자에게 보고하고 중단. 자동 복구하지 않는다.
   - 둘 다 없음 → 첫 적용. 1.4 에서 append.

### 1.3 프리셋 선택 (인자 없을 때)

`AskUserQuestion` 한 번으로 프리셋 선택. **단일 선택**, 옵션은 `§3` 의 4종 + "제거" + "취소".

```text
질문: "프로젝트 CLAUDE.md 에 어떤 문체 지침을 박을까요? (현재: <개조식|평어체|... |없음>)"
옵션:
  1. 개조식        — 명사형 종결, 한 줄 한 항목, 동사 최소
  2. 평어체        — "~다" 평서체, 존댓말/반말 모두 X
  3. 기획보고서    — 두괄식, 배경·현황·방안 구조, 표·번호 적극
  4. 규정          — 조·항·호 번호, 의무/임의/금지 어미 구분
  5. 제거          — marker 구간 통째 삭제
  (취소는 시스템 옵션)
```

`(현재: …)` 자리에는 1.2 에서 추출한 프리셋 헤더(예: `**문체 — 평어체 (set_style)**`)에서 이름만 뽑아 표시.

### 1.4 marker 구간 갱신

선택된 프리셋의 본문(§3) 을 가져와 다음 규칙으로 `CLAUDE.md` 에 반영한다.

| 현재 상태 | 동작 |
|:---|:---|
| 파일 없음 | `# CLAUDE.md` heading + 빈 줄 + marker 영역(begin/end + 본문) 으로 신규 생성 |
| 파일 있고 marker 없음 | 파일 끝에 빈 줄 두 줄 + marker 영역 **append** |
| 파일 있고 marker 양쪽 있음 | **두 marker 사이만** 새 프리셋 본문으로 교체 (marker 줄 자체는 유지) |
| 파일 있고 marker 한 쪽만 | **중단** 후 사용자 보고 |

marker 영역 형식:

```markdown
<!-- STYLE-BEGIN /set_style -->
{프리셋 본문 — §3 의 해당 항목을 글자 그대로}
<!-- STYLE-END /set_style -->
```

### 1.5 사후 보고

한 묶음으로 사용자에게 출력:

```text
[set_style]
  target  = CLAUDE.md
  이전    = <개조식|평어체|기획보고서|규정|없음>
  적용    = <개조식|평어체|기획보고서|규정|제거됨>
  marker  = <created|appended|updated|removed>
  본문크기 = <bytes>
다음 세션부터 위 문체로 응답합니다.
```

## 2. `remove` 동작

- marker 양쪽이 모두 있어야 안전 제거 가능. 없으면 "이미 적용된 문체 없음" 출력 후 종료.
- 제거 시 marker 두 줄과 그 사이 본문, 그리고 직전 빈 줄까지만 삭제한다. marker 위·아래의 사용자 수동 내용은 보존.

## 3. 프리셋 본문 (글자 그대로 marker 사이에 쓴다)

> 본문은 모두 blockquote(`>`) 로 시작한다. CLAUDE.md 안에서 시각적으로 분리되며, Claude 가 매 세션 자동으로 인지한다.

### 3.1 개조식

```markdown
> **문체 — 개조식 (set_style)**
> 사용자에게 보내는 모든 텍스트(채팅 응답·보고·요약) 는 다음 규칙을 따른다.
> - 명사형 종결로 끝낸다 — "~함", "~임", "~예정", "~필요", "~중".
> - 한 항목은 한 줄. 동사·조사 최소화, 핵심 명사 위주로 압축.
> - 계층은 들여쓰기·번호(1./1.1/가./나.) 로 표현.
> - 완전한 서술 종결("~다", "~합니다") 회피.
> - 단, 코드·명령·파일 경로·인용·에러 메시지는 원문 그대로 보존.
> - 사용자가 명시적으로 "풀어서 설명" 을 요청하면 그 응답 한 번만 평어체 허용.
```

### 3.2 평어체

```markdown
> **문체 — 평어체 (set_style)**
> 사용자에게 보내는 모든 텍스트는 다음 규칙을 따른다.
> - "~다" 평서체로 종결한다 — "~한다", "~이다", "~된다", "~없다".
> - 존댓말("~합니다", "~입니다") 과 반말("~해", "~야") 모두 사용하지 않는다.
> - 신문기사·기술문서·논문에서 쓰는 중립 서술체를 유지한다.
> - 의문문은 "~인가", "~는가" 형식.
> - 코드·명령·파일 경로·인용·에러 메시지는 원문 그대로 보존.
```

### 3.3 기획보고서

```markdown
> **문체 — 기획보고서 (set_style)**
> 사용자에게 보내는 모든 텍스트는 다음 규칙을 따른다.
> - 두괄식: 핵심 결론·권장안을 가장 먼저 한 문단(또는 한 줄) 으로 제시한다.
> - 구조: 배경 → 현황 → 과제 → 방안 → 기대효과 순서를 기본으로 한다 (필요한 절만 사용).
> - 본문은 명사형 개조식 위주, 보조 설명만 평어체("~다") 로 혼용.
> - 표·번호 목록을 적극 사용하고 산문 단락은 최소화한다.
> - 정량 수치·근거·일정을 명시한다 — "3건", "2026-05-30 까지", "약 12% 감소".
> - 추측·가정에는 "~로 추정", "~할 가능성" 등 한정 표현을 붙인다.
```

### 3.4 규정

```markdown
> **문체 — 규정 (set_style)**
> 사용자에게 보내는 모든 텍스트는 다음 규칙을 따른다.
> - 조·항·호 체계로 번호를 부여한다 — 제1조, ①, 1., 가.
> - 의무·강제·임의·금지를 어미로 구분한다:
>   - "~한다" (의무 / 원칙)
>   - "~해야 한다" (강제)
>   - "~할 수 있다" (임의 / 허용)
>   - "~하지 아니한다" (금지)
> - 정의·적용범위·예외를 명시적으로 분리한다 — "정의:", "적용범위:", "단, ~한 경우는 제외한다".
> - 한 조항은 한 의무만 다룬다 — 복합 의무는 항(①②③) 으로 분리.
> - 줄임말·이모지·구어체("~인 듯", "~같아요", "ㅋ") 금지.
> - 인용·코드·파일 경로는 원문 그대로 보존.
```

## 4. DO / DON'T

### DO

- 호출마다 marker 사이를 **완전 교체** 한다.
- marker 가 한 쪽만 있으면 **중단** 하고 사용자 보고한다 (자동 복구 금지).
- 프리셋 본문은 §3 텍스트를 **글자 그대로** 복사한다.
- 사후 보고에 이전 → 적용 프리셋, marker 변화(`created|appended|updated|removed`) 를 명시한다.

### DON'T

- 사용자 전역 `~/.claude/CLAUDE.md` 를 수정하지 않는다.
- marker 바깥 사용자 수동 영역을 절대 손대지 않는다.
- 사용자가 "이번 답변만 다르게" 라고 해도 marker 본문은 수정하지 않는다 — 일회성 변형은 그 응답 안에서만 적용.
- 프리셋 본문을 임의로 줄이거나 풀어쓰지 않는다 (드리프트 방지).
- 새 프리셋을 즉흥 추가하지 않는다 — 추가가 필요하면 본 SKILL.md 의 §3 에 정식 항목으로 등록 후 옵션에 노출.

## 5. 체크리스트

- [ ] `$CLAUDE_PROJECT_DIR/CLAUDE.md` 가 대상 — 전역 CLAUDE.md 손대지 않음.
- [ ] marker 양쪽 검사 → 한쪽만 있으면 중단·보고.
- [ ] 인자 없으면 `AskUserQuestion` 한 번으로 프리셋 선택받음 (4 + 제거 + 취소).

- [ ] marker 바깥 영역 변경 없음 (diff 로 확인).
- [ ] 사후 보고 한 묶음 출력 (이전·적용·marker 상태·본문크기).

## 6. 예시

### 6.1 처음 적용 (CLAUDE.md 존재, marker 없음)

호출: `/set_style`
선택: `평어체`

결과 ([CLAUDE.md](../../../CLAUDE.md) 끝에 append):

```markdown
(기존 내용)


<!-- STYLE-BEGIN /set_style -->
> **문체 — 평어체 (set_style)**
> 사용자에게 보내는 모든 텍스트는 다음 규칙을 따른다.
> - "~다" 평서체로 종결한다 …
…
<!-- STYLE-END /set_style -->
```

보고:
```
[set_style]
  target  = CLAUDE.md
  이전    = 없음
  적용    = 평어체
  marker  = appended
  본문크기 = 412B
```

### 6.2 교체 (marker 있음)

호출: `/set_style 기획보고서`

동작: 두 marker 사이만 §3.3 본문으로 교체. marker 줄 자체는 그대로.

보고:
```
[set_style]
  target  = CLAUDE.md
  이전    = 평어체
  적용    = 기획보고서
  marker  = updated
  본문크기 = 538B
```

### 6.3 제거

호출: `/set_style remove`

동작: marker 두 줄 + 본문 + 직전 빈 줄까지 삭제. 다른 영역 보존.

보고:
```
[set_style]
  target  = CLAUDE.md
  이전    = 기획보고서
  적용    = 제거됨
  marker  = removed
```

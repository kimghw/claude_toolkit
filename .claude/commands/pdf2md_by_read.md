# /pdf2md_brief 명령

인자: $ARGUMENTS

## 목적

PDF를 **챕터 경계 우선**으로 분할하고(챕터가 길면 20p 단위로 하위 분할), **청크 수만큼 서브에이전트를 병렬 기동**하여 각 에이전트가 **Read 도구로 담당 페이지를 직접 읽어** 무손실 마크다운으로 변환한 뒤, 결과를 **페이지 경계 형식 일치를 검토하며 병합**한다. `pdf2md` 스킬의 큐·락·물리분할(qpdf)·세션 격리 없이 한 세션에서 가볍게 처리하는 경량 변형이다.

> 대용량(수백 페이지)·다중 파일·세션 간 재개·중복 방지가 필요하면 본 명령 대신 `pdf2md` 스킬을 사용한다.

## 분할 우선순위 (핵심)

1. **챕터 경계 우선(최우선)**: 가능하면 **장(chapter) 단위로 분할**한다. 1개 챕터 = 1개 청크가 기본이며, 서로 다른 챕터를 한 청크에 섞지 않는다. 챕터 경계에서 끊으면 헤딩·문맥이 온전히 보존되어 변환·병합 품질이 높다.
2. **20p 상한(제약)**: Read 도구는 PDF를 **1회 호출당 최대 20페이지**까지 읽는다. 따라서 한 챕터가 20페이지를 초과하면 **그 챕터 내부에서만** 20p 이하 하위 청크로 다시 나눈다(챕터 경계는 유지). 각 서브에이전트는 담당 구간을 **단일 Read 호출**로 직독한다.
3. **폴백(균등 20p)**: 아웃라인(북마크)·목차로 챕터 경계를 식별할 수 없으면 20p 균등 분할로 폴백하고, 그 사실을 사용자에게 보고한다.

이 우선순위가 "read 도구 직독" + "챕터별 분할" 원칙의 근거다(본문 사전 추출·qpdf 물리 분할 불필요).

## 인자 처리

| 인자 | 동작 |
|---|---|
| `help` / `-h` / `--help` | 본 명령 사용법을 한눈에 출력하고 종료 |
| `<pdf경로>` | 해당 PDF를 변환 |
| (없음) | 변환할 PDF 경로를 사용자에게 요청 |

## 절차

### 0. 입력 확인

- `$ARGUMENTS`가 비어 있으면 변환할 PDF 경로를 사용자에게 묻는다.
- `test -f <pdf>`로 존재 여부를 확인한다. 없으면 보고 후 종료.

### 1. 사전 청킹·구조 검토 (변환 전 — 필수)

> 원본 요구사항 ⑤ "페이지 변환 전에 어떻게 청킹하고 어떻게 구조를 가져갈지 검토" + 챕터 경계 우선.

- `pdfinfo <pdf>`로 **총 페이지 수**를 조회한다.
- **챕터 경계 탐지(최우선)** — 아래 순서로 시도한다.
  1. **아웃라인/북마크**: 사용 가능한 도구로 챕터 제목·시작 페이지를 추출한다. 예) `mutool show <pdf> outline`, `pdftk <pdf> dump_data | findstr Bookmark`, 또는 Python `pypdf`(`reader.outline`).
  2. **목차(TOC) 직독**: 아웃라인이 없으면 앞부분 목차 페이지를 **Read로 직접 읽어** 장 제목과 시작 페이지를 파악한다.
  3. 둘 다 불가하면 **20p 균등 분할**로 폴백(사용자에게 명시).
- **청크 산출**:
  - 각 챕터를 `[시작페이지, 끝페이지]` 구간으로 만든다(다음 챕터 시작 직전까지).
  - 구간 페이지 수 > 20이면 **해당 챕터 내부에서** 20p 이하 하위 청크로 분할한다(예: 35p 챕터 → `1-20`, `21-35`).
  - 결과: 청크 리스트(각 청크 = `{챕터명, part_index, 페이지범위, 챕터내_순번}`), 각 범위 ≤ 20p.
- **구조 사전 검토**: 헤딩 체계(장·절 번호 규칙)와 최상위 제목 → `#`~`######` 레벨 매핑, 표·이미지·수식·각주(첨자) 유무를 파악한다. 모든 서브에이전트가 공유할 **공통 변환 규약**(헤딩 레벨 매핑·번호 표기 보존·페이지 마크 금지·무손실)을 1개로 확정한다(일관성 단일 출처).
- 청크 계획(챕터별 청크 수·각 페이지 범위·공통 규약·폴백 여부)을 사용자에게 간단히 보고한다.

### 2. 멀티에이전트 병렬 변환

> 원본 요구사항 ①②③ "분할 · 청크 수만큼 멀티에이전트 · Read 도구 직독".

- 작업 폴더를 준비한다: `mkdir -p <원본폴더>_md/.brief_parts/<stem>/` (`<stem>` = 확장자 제외 파일명).
- **청크 수만큼 Agent를 단일 메시지에서 병렬 기동**한다 (`subagent_type: "general-purpose"`, `run_in_background: true`). 각 에이전트는 정확히 1개 청크만 담당한다.
- 각 에이전트 프롬프트에 다음을 주입한다.
  - 원본 PDF **절대경로**.
  - 담당 **챕터명**과 페이지 범위 `<start>-<end>` → **반드시 Read 도구의 `pages` 인자로 직독**(본문을 미리 텍스트로 추출해 넘기지 않는다).
  - 챕터 내 하위 청크인 경우 "챕터 N의 a/b번째 구간"임을 알려 헤딩 중복 부여를 방지한다(같은 챕터 제목을 하위 청크마다 다시 달지 않음).
  - 1절에서 확정한 **공통 변환 규약**(전 에이전트 동일 문자열).
  - 출력 파일 경로 `<원본폴더>_md/.brief_parts/<stem>/part<NN>.md` (`<NN>` = 읽기 순서 2자리 zero-pad).
  - 무손실 규칙: 원문 텍스트 삭제·요약·의역 금지, 페이지 경계 마크(`--- page N ---` 등) 삽입 금지, 잘린 문장 임의 완성 금지(병합 단계에서 처리).
  - (이미지 포함 PDF인 경우) `pdfimages -all -f <start> -l <end> <pdf> <원본폴더>_md/assets/<stem>/part<NN>-fig`로 담당 구간 이미지를 추출하고 `![](assets/<stem>/part<NN>-fig-XXX.ext)` 상대 링크로 삽입. 추출 이미지 수를 완료 보고에 포함.
- 완료 보고에 **담당 챕터·범위 · 변환한 헤딩 목록 · 마지막 줄 상태(문단/목록/표 잘림 여부) · 추출 이미지 수**를 포함하도록 요청한다.

### 3. 병합 + 페이지 경계 형식 검토

> 원본 요구사항 ④ "md 변환 후 병합할 때 페이지 간 형식이 맞는지 검토".

- 모든 서브에이전트 완료 후 `part01.md`부터 순서대로 연결하여 `<원본폴더>_md/<stem>.md`로 병합한다.
- **경계 형식 일치 검토**(각 `partNN.md` ↔ `part(NN+1).md`):
  - **같은 챕터를 분할한 경계**(하위 청크 사이): 문단·목록·표가 잘렸으면 이어붙여 구조를 복원한다.
    - 문단 단편 → 한 칸 공백으로 연결.
    - 목록 계속 → 목록 구조 하나로 복원, 번호 연속성 확인.
    - 표 계속(헤더 없이 시작하는 행) → 앞 파트 표에 병합.
  - **챕터 경계**(서로 다른 챕터 사이): 보통 새 헤딩으로 자연 분리되므로 이어붙이지 않는다. 챕터 제목 헤딩 레벨이 일관적인지만 확인한다.
  - 헤딩 레벨이 청크마다 다르게 부여됐는지(같은 깊이인데 레벨 불일치) 점검 → 1절 공통 규약 기준으로 통일.
  - **내용 자체는 변경하지 않고 구조·레벨만 복원**한다.
- 첨자(상·하첨자 등 HTML 태그) 사용 파트가 하나라도 있으면 병합 파일 최상단에 `<!-- markdownlint-disable MD033 -->`를 1회 주입한다.
- 이미지가 있으면 링크가 `<원본폴더>_md/<stem>.md` 기준 올바른 상대경로(`assets/<stem>/...`)를 가리키는지 확인한다.

### 4. 검증 · 정리 · 보고

- `markdownlint <원본폴더>_md/<stem>.md` 실행(프로젝트에 `.markdownlint.json` 있으면 우선). 위반은 구조·포맷 범위에서 자가 수정 후 재검증(원문 의미 불변).
- **헤딩 순서·계층·표기**가 원문 PDF의 장·절 순서와 일치하는지(누락·중복·역전·레벨 점프 없음) 스폿 체크.
- 이미지 링크가 있으면 `test -f`로 실제 파일 존재를 확인한다(이미지 0개면 "이미지 없음"으로 명시).
- 검증 통과 후 `<원본폴더>_md/.brief_parts/<stem>/`를 정리(`rm -rf`)한다. `<stem>.md`와 `assets/`는 보존.
- 사용자에게 **최종 경로 · 총 페이지 · 챕터/청크 수 · 폴백 여부 · 병합 경계 이어붙인 곳 · 수정한 lint 항목**을 요약 보고한다.

## DO / DON'T

**DO**

- **챕터 경계를 최우선으로** 분할하고, 챕터가 20p를 넘을 때만 그 안에서 20p 이하로 하위 분할한다.
- 변환 **전에** 청크 계획과 공통 변환 규약을 확정해 전 에이전트에 동일하게 주입한다(일관성 단일 출처).
- 청크 수만큼 에이전트를 **단일 메시지에서 병렬** 기동한다.
- 각 에이전트는 **Read의 `pages` 인자로 담당 구간(≤20p)을 직독**한다.
- 병합 시 같은 챕터를 나눈 경계의 문단·목록·표 잘림을 판정해 구조만 복원한다.

**DON'T**

- **서로 다른 챕터를 한 청크에 섞지 않는다**(챕터 경계 우선).
- 본문을 사전 텍스트화(`pdftotext`·`pymupdf` 등)하여 에이전트에 문자열로 넘기지 않는다.
- 청크당 20페이지를 초과해 Read 호출하지 않는다(도구 상한).
- 페이지 경계 마크를 중간·최종 산출물에 삽입하지 않는다.
- 한 에이전트에 2개 이상 청크를 맡기지 않는다.
- 대용량·다중 파일·세션 재개가 필요한 작업을 본 경량 명령으로 처리하지 않는다(→ `pdf2md` 스킬 사용).

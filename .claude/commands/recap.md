# /recap 명령

인자: $ARGUMENTS

## 목적

이번 세션(현재 대화) 동안 사용자가 내린 **지시·요구사항만** 추려서 한 파일로 정리한다. Claude의 답변·도구 호출·중간 상태는 제외한다.

## 사전 참고

- 프로젝트 루트의 `CLAUDE.md`를 먼저 읽어 저장 위치·언어·기타 규칙을 확인한다.
- 현재 프로젝트의 경우 사용자 요구사항은 `user_requirements/` 폴더에 저장한다.

## 동작 규칙

1. **대화 스캔**:
   - 현재 세션의 user 메시지만 시간순으로 훑어 "지시·요구사항"에 해당하는 항목을 추출.
   - 단순 질문·잡담·확인성 답변("응", "ok", "그래")은 제외.
   - 같은 주제로 이어진 지시는 하나로 합치고, 사용자가 도중에 방향을 바꿨다면 최종 결론만 남긴다.

2. **파일 작성**:
   - 저장 위치: `user_requirements/recap_<YYYY-MM-DD>.md` (오늘 날짜 기준, CLAUDE.md의 `currentDate`가 있으면 그 값을 사용).
   - 같은 날짜 파일이 이미 있으면 뒤에 `_2`, `_3` 식으로 붙여 새 파일을 만든다 (덮어쓰지 않음).
   - 형식:
     ```markdown
     # 세션 지시 정리 — <YYYY-MM-DD>

     ## 요약
     <2~3줄로 이번 세션의 주된 요청 한눈 보기>

     ## 지시 목록
     1. <지시 1 — 한 줄 요약>
        - 상세: <필요시 부가 설명>
        - 결과: <완료 / 진행 중 / 보류>
     2. ...

     ## 후속 작업
     - <남은 일이 있으면 bullet로>
     ```
   - 언어는 한국어로 작성.

3. **인자 처리**:
   - 인자가 `help` / `-h` / `--help`이면 본 명령이 받는 인자 목록(없음/파일경로/`dry`·`preview`)과 동작 설명을 출력하고 종료.
   - 인자가 비어 있으면 위 기본 동작 수행.
   - 인자가 파일명/경로면 그 경로에 저장.
   - 인자가 `dry` 또는 `preview`면 파일을 만들지 않고 정리 결과만 출력.

4. **마무리**:
   - 저장한 파일 경로를 마크다운 링크로 출력.
   - 누락 가능성이 있는 항목이 있으면 한 줄로만 알려준다.

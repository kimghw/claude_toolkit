# /recap 명령

인자: $ARGUMENTS

## 목적

이번 세션에서 사용자가 내린 **지시·요구사항만** 추려서 **테마별 파일**(`user_requirements/recap_<theme>.md`)로 정리한다. 동시에 사용자의 원문 발화(질문·지시)만 모은 단일 누적 파일 `user_requirements/user_questions.md`에 새 항목을 추가한다 — 이 파일에는 날짜·시간이 들어가지 않는다. 정리 시 **기존 요구사항 문서와의 일관성**과 **세션 내 지시들 간의 논리성**을 함께 점검한다. 실제 정리·점검·저장은 `recap-worker` 서브에이전트가 수행하고, 슬래시 커맨드는 컨텍스트 추출과 디스패치만 담당한다.

## 동작 규칙

1. **인자 처리**:
   - `help` / `-h` / `--help` → 본 명령의 인자(없음 / `dry`·`preview`)와 동작 설명을 출력하고 종료. 서브에이전트 호출하지 않음.
   - 비어 있음 → 기본 저장 모드. 서브에이전트가 세션 지시들의 주된 테마(들)를 추출해 테마별로 `recap_<theme>.md` 파일을 만들고(이미 있으면 병합), 동시에 `user_questions.md`에 사용자 원문 발화를 누적 추가한다. 파일명에는 날짜를 포함하지 않는다.
   - `dry` 또는 `preview` → 저장하지 않고 결과만 출력하는 모드(테마별 본문과 추가 예정 question 항목을 화면에 미리 보여줌).
   - 그 외 인자는 무시(과거 "파일경로 지정" 모드는 폐지 — 파일명은 항상 테마로 결정).

2. **사전 준비 (오케스트레이터 직접 수행)**:
   - 프로젝트 루트의 `CLAUDE.md`를 Read로 열어 저장 위치 규칙, 언어 규칙을 확인한다.
   - 현재 세션의 user 메시지를 시간순으로 훑어 두 가지 산출물을 만든다:
     - `user_instructions`: 잡담·확인성 응답("응", "ok", "그래")을 제거하고, 같은 주제 지시는 합치고, 도중 방향이 바뀌었다면 최종 결론만 남긴 정제 목록(시간순 번호). 방향 전환 사실은 별도 메모로 남겨 서브에이전트에 함께 전달.
     - `raw_user_messages`: **필터 전** 사용자 발화 원문 리스트. 잡담·확인성 응답·tool 출력은 제거하되 사용자 본인의 질문/지시는 가공 없이 그대로. 어떤 형태로도 날짜·시간 메타를 포함하지 않는다(서브에이전트가 `user_questions.md`에 그대로 누적 append 할 입력).

3. **서브에이전트 위임**:
   - `Agent` 도구를 `subagent_type: recap-worker`로 호출한다.
   - `description`은 짧게 "Session recap 작성"으로.
   - `prompt`에 다음 항목을 명시적으로 포함한다(서브에이전트는 세션 컨텍스트를 보지 못한다):
     - `project_root`: 현재 작업 디렉토리 절대경로
     - `mode`: `save` 또는 `dry`
     - `claude_md_excerpt`: CLAUDE.md에서 저장 규칙 관련 부분 발췌(원문 인용)
     - `user_instructions`: 위에서 추출한 정제된 지시 목록(시간순 번호). 각 항목은 "원문 요약 + (필요 시 방향 전환 메모)" 형태.
     - `raw_user_messages`: 위에서 추출한 사용자 발화 원문 리스트(날짜·시간 제거 상태).
   - 서브에이전트가 테마 결정 → 일관성·논리성 점검 → 테마별 파일 저장(병합) → `user_questions.md` 누적 갱신을 단독으로 수행한다.

4. **마무리**:
   - 서브에이전트의 `완료_보고` YAML을 받아 사용자에게 다음을 출력한다:
     - 저장/병합된 테마 파일 경로 목록(마크다운 링크)
     - `user_questions.md`에 추가된 항목 수
     - `강조_알림`이 "없음"이 아니면 ⚠ 아이콘과 함께 한 줄로 강조
     - 일관성/논리성 카운트 한 줄 요약(예: "확장 3, 신규 2, 충돌 0 / 미해결 의존 1")
   - dry 모드면 본문도 그대로 사용자에게 전달.

## 위임 시 주의

- 서브에이전트는 **현재 세션 대화 내용을 볼 수 없다**. user 메시지 추출은 반드시 오케스트레이터가 prompt에 담아 전달해야 한다.
- 서브에이전트는 기존 `user_requirements/*.md`를 자체적으로 Read하므로 그 내용을 prompt에 다시 담을 필요는 없다.
- 동일 테마 파일이 이미 있으면 서브에이전트가 새 내용을 해당 파일에 **병합**(기존 절에 추가)한다. 같은 이름으로 `_2`, `_3` 식 새 파일을 만들지 않는다.
- `user_questions.md`는 **append-only 누적 파일**이며 날짜·시간 메타를 포함하지 않는다. 이미 존재하는 문장과 동일한 항목은 중복 추가하지 않는다.


# 사용자의 질의는  <cwd>/interactive 폴더에 주제/기능 별로 정리해서 저장 후 관리 한다. 
- 1개 질문에 대해서 답변이 완료하면 업데이트 한다. 
- 업데이트 시, 서브에이전트를 이용해서 기존 파일과 일관성, 논리성 등을 검토한다.  
  
# 사용자는 툴이나 요구사항을 만들기 직전에 <cwd>/spec 에 사전에 요구사항을 정리해서 만들 수 있다. 
- 네이밍 규칙은 접두사로 spec_***.md 로 한다.

# 지식저장소 <cwd>/context 를 두고 자료조사나 프로젝트 진행 중 알게된 사실에 대해서 관리한다.
- 사용자 요청시 관련자료가 있는지 우선적으로 참조한다.
- 새로운 지식이 발견된 경우, 서브에이전트를 이용해서 갱신하고 필요한 경우, 기존 내용과 일관성, 논리성을 검토한다.

# 아이디어나 자료 조사는 개조식, 평어체를 사용하고, 보고서를 작성할 경우 보고서 스타일을 사용할 것
